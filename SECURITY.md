# 安全策略（Security Policy）

## 支持的版本

| 版本 | 状态 |
|------|------|

## 报告漏洞

如果你发现了安全漏洞，**请不要通过公开 Issue 披露**。请通过以下方式私密报告：

- GitHub：在仓库中提交 **Private vulnerability report**（Security → Report a vulnerability）
- 或发送邮件至项目维护者，并在标题注明 `[SECURITY]`

请尽量提供：

- 漏洞类型与影响面
- 复现步骤 / PoC
- 受影响版本与运行环境

我们会在收到后 72 小时内确认，并协商修复与披露时间表。

## 安全注意事项

- 本服务默认监听 `0.0.0.0` 且 **未内置鉴权**。部署到公网前请务必：
  - 置于反向代理 / 网关之后并启用鉴权（如 API Key、mTLS）；
  - 或用防火墙限制来源 IP；
  - 不要将 `.env`（含内网地址、模型路径等）提交进仓库。
- 裁剪图以 base64 内联返回，处理敏感文档时注意响应可能包含完整内容，避免被日志/中间件落盘。
- 任务队列在内存中保存结果，重启即丢失，不应用于持久化敏感数据。
